**Generated**: 2026-05-15
**Spec ref**: R-CSSE-5, Q3, AC12
**Amendment 1 note**: chunk-level tracking deferred to cycle 5+ (Opt C). Effective sizes
computed from stub + transitive chunk union (metafile-derived, no stable-key tracking needed).

---

## Methodology

**Effective consumer size** for a set of subpaths = sum of stub file sizes + union of all
chunks imported by those subpaths. The union de-duplicates shared chunks, which is the
mechanism by which splitting reduces bytes for multi-subpath consumers.

- **v2.8.0 comparison**: `totalBytes` from `reports/bundle/v2.8.0-baseline.json` (schema v2).
  With `splitting: false`, each subpath file was monolithic (all dependencies inlined).
- **v2.8.1 measurement**: stub file size + transitive chunk union (sizes from analyze build
  metafile `dist/metafile-esm.json`, esbuild-reported chunk sizes).
- **Bundler assumption**: consumer uses webpack or rollup (honors `sideEffects`).
  esbuild / vite consumers do NOT benefit from `sideEffects`; chunk byte reductions still apply
  but tree-shaking from the `sideEffects` field does not. See CHANGELOG for disclosure.

## Notes

- **esbuild / vite consumers**: chunk byte reductions apply (shared code loaded once),
  but `sideEffects` tree-shaking does NOT apply. esbuild ignores `package.json sideEffects`.
  webpack and rollup consumers benefit from BOTH chunk splitting AND `sideEffects`.
- **Cycle 5 outlook**: the 3 singleton files (`utils.es6.js`, `popup.es6.js`, `tooltip.es6.js`)
  remain in the `sideEffects` array because their constructors run side effects at import time.
  Cycle 5 (singleton lazy-init refactor) will remove them from the array, unlocking additional
  tree-shaking savings for esbuild/vite consumers.
- **Chunk-level tracking**: chunk files are produced in `dist/chunks/` but are not tracked
  individually in `v2.8.1-baseline.json` (Opt C deferral, Amendment 1). Effective sizes in
  this report are computed from the analyze-build metafile, not the baseline JSON.
- **SC-A near-threshold note**: SC-A's 19.8% saving barely clears the ≥19% threshold. This is
  because popup and form share only 3 of their chunks (popup uses 4, form uses 8). A cycle 5
  singleton refactor would extract more shared dependencies and improve this ratio further.
